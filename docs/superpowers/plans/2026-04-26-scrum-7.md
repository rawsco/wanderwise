# SCRUM-7 — Allow typing dates in the trip form

## Problem

In `TripForm.tsx`, start/end dates use a native `<input type="date">` via `register("startDate"|"endDate")`. On many browsers/locales, native date inputs split into rigid DD/MM/YYYY segments — typing the digits `01051979` does not advance through fields, the DD slot just keeps getting overwritten. Users who prefer keyboard entry are blocked.

The rest of the app (StopList, StopSearch) already uses the custom button-based `DatePicker` (`src/components/ui/date-picker.tsx`), but it has no text-typing affordance either — it only opens a calendar.

## Goal

Replace the trip start/end date inputs with a control that supports **both** modes (per the ticket: "Some users like to use a date picker some like to just type in the date numbers"):

1. A text input that accepts digits and auto-formats to `DD/MM/YYYY`
2. A calendar icon button that opens a `DayPicker` popover (same UX as the existing `DatePicker`)

## Scope

- New: `src/components/ui/date-input.tsx` — typeable + calendar combo
- Modify: `src/components/trip/TripForm.tsx` — use the new component for start/end dates via `Controller`

Out of scope: changing other date inputs (StopList/StopSearch use the existing `DatePicker` and are not the bug site).

## Design — `DateInput`

**Public interface** matches `DatePicker` to keep substitutability:

```ts
interface DateInputProps {
  value?: string;          // ISO YYYY-MM-DD
  onChange: (value: string) => void;
  min?: string;            // ISO
  max?: string;            // ISO
  disabledRanges?: DateRange[];
  placeholder?: string;    // default "DD/MM/YYYY"
  className?: string;
  id?: string;
}
```

**Layout:**
- Outer wrapper `relative min-w-0`
- A flex row with:
  - `<input type="text" inputMode="numeric" maxLength={10}>` taking the full width, `text-base` for iOS, `min-w-0 w-full`
  - A right-aligned calendar icon button (same look as `DatePicker`'s trigger but icon-only) that opens the popover
- Popover: portal-rendered `DayPicker`, positioned via `getBoundingClientRect` — copy-paste the popover/positioning code from `date-picker.tsx` rather than reinvent it.

**Typing behavior:**
- Maintain a local `display` string ("01/05/1979" form) so the user sees what they typed.
- Strip non-digits on input. After every keystroke, re-format: insert `/` after position 2 and after position 5 (so digits "01051979" render as "01/05/1979" mid-typing).
- Allow backspacing through separators naturally (we just strip and reformat — caret position isn't critical for this fix).
- When `display` parses to a valid date in `DD/MM/YYYY` form: emit `onChange(formatISO(date))`.
- When `display` is empty: emit `onChange("")`.
- When `display` is partial / invalid: do nothing (don't emit). On blur, if it looks like an attempted date but invalid, leave as-is — form-level Zod validation will catch the missing/invalid value.
- When the parent's `value` prop changes (e.g., picker selection or initial defaultValues), sync `display` from it.

**Picker behavior:**
- Clicking the calendar icon opens the `DayPicker` popover (re-use the positioning/portal/escape/outside-click handlers from `date-picker.tsx`).
- Selecting a day: emit ISO via `onChange`, sync `display` to formatted form, close popover.
- Pass through `min`/`max`/`disabledRanges` to `DayPicker` matchers using the same helper logic as `date-picker.tsx`.

**iOS rules (per CLAUDE.md):**
- `text-base` on the input (16px) — prevents auto-zoom.
- `min-w-0 w-full` on input, `min-w-0 overflow-hidden` on the wrapping container — required for date/time inputs.

## Design — `TripForm` integration

- Import `Controller` from `react-hook-form`.
- Replace each `<Input type="date" {...register(...)}>` with:

```tsx
<Controller
  name="startDate"
  control={control}
  render={({ field }) => (
    <DateInput id="startDate" value={field.value} onChange={field.onChange} />
  )}
/>
```

- Destructure `control` from `useForm()`.
- Keep error display (`errors.startDate?.message`) untouched.
- Keep the existing `endDate < startDate` cross-field check in `onSubmit`.

## Manual verification (subagent / human reviewer)

- `/trips/new`: type `01051979` into Start date — expect `01/05/1979` to appear and ISO `1979-05-01` stored.
- Click calendar icon: popover opens; select a day; input updates.
- Submit with Start = today, End = tomorrow → succeeds.
- Submit with End < Start → existing error fires.
- `/trips/[id]/edit`: existing dates render in DD/MM/YYYY form on load.
