**Goal:** When the trip detail page loads or the stop list changes, the map automatically zooms and pans to fit every stop in the route, so the user sees the whole trip without manual zooming.

**Ticket:** https://devotonomy.atlassian.net/browse/SCRUM-8
**Branch:** `scrum-8`

## Background

`src/components/trip/TripMap.tsx` currently uses `defaultCenter={stops[0]}` and `defaultZoom={stops.length > 1 ? 6 : 5}`. Because `defaultCenter` / `defaultZoom` only apply on first mount, adding or removing stops never re-centres the map, and the hardcoded zoom does not account for the actual geographic spread (a route from Edinburgh to London ends up the same zoom as a route across Australia).

`@vis.gl/react-google-maps` exposes a `useMap()` hook that returns the underlying `google.maps.Map` once it has loaded. From there, `map.fitBounds(bounds, padding)` is the standard Google Maps mechanism to frame a list of points.

## Approach

Add a small child component, `FitToStops`, rendered inside `<Map>`. It subscribes to the map instance via `useMap()` and runs an effect whenever the stops' coordinates change:

- 0 stops â†’ no-op (the existing UK-fallback default still applies).
- 1 stop, or all stops at the same place (start â‰ˇ end with no intermediates) â†’ `setCenter` to that point and `setZoom(12)` (a sensible "single location" zoom; `fitBounds` on a degenerate bounds would zoom to street level).
- 2+ stops at different positions â†’ build a `google.maps.LatLngBounds`, extend with each stop's lat/lng, and call `fitBounds(bounds, 64)` for ~64px of inset on each edge.

The effect's dependency array uses a string key derived from the stops' lat/lng (rounded), not the stops array reference. This means non-position edits (date change, booking-status change) do not trigger a re-fit and so do not snap the user out of any pan/zoom they have done manually.

The existing `defaultCenter` / `defaultZoom` props are kept as the pre-load fallback paint; once the map loads, `FitToStops` takes over within the same tick.

## Validation


Then load the trip detail page in the dev stack and verify:

- Trip with stops across the UK frames the whole UK, not just one point at zoom 6.
- Adding a stop in a new region re-frames to include it.
- Editing dates / booking status does not re-frame.
- A trip with a single stop centres on that stop at a reasonable zoom.
- A trip with no stops still renders the default UK view.
